# Sprint 3: Advanced Operations (SHOULD Tier)

**Goal:** Implement the complex, advanced tier features including SLA policies, dashboard metrics, activity logs, and ticket assignment to pull ahead.

### Scope & Issues
- [x] Create `ActivityLog` model and migration.
- [x] Integrate SLA timers on ticket creation (`sla_breach_at`) based on priority.
- [x] Build `DashboardController` for fetching global workspace metrics.
- [x] Update React UI to include a metrics bar (Total, Open, Urgent, SLA Breached).
- [x] Add Activity Log / Audit Trail to the ticket detail page.
- [x] Add "Claim Ticket" functionality for agents.

### Notes & Decisions
- Activity logs are automatically generated by the `TicketController` during lifecycle events (create, assign, comment) to ensure they are tamper-proof.
- SLA timers are derived purely backend-side to prevent client spoofing.
- The React UI was updated to a grid layout to elegantly fit the conversation and activity timeline side-by-side.

### Outcome
We've successfully elevated the platform from a basic ticketing app to a proper helpdesk capable of tracking metrics, assignments, and audit trails.
